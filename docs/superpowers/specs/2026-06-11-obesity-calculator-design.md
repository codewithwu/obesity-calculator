# 2026 肥胖评估工具 — 设计文档

- **日期**：2026-06-11
- **状态**：已批准
- **范围**：单次实现周期内的完整功能

## 1. 目标与原则

构建一个轻量、面向亚裔（默认）用户的中文 Web 应用，让用户在 3 个字段内完成"是否肥胖"的自我评估，同时提供 2026 年 ADA 新标准的科普阅读入口。GitHub Pages 静态部署，移动端 / 桌面端共用同一份代码。

**原则**：

- 测算路径零摩擦：默认 3 字段，3 秒出结果
- 不"医疗化"：用温和的视觉与文案，避免制造焦虑
- 解读内容独立成页，不打扰测算流程
- 单一语言（中文）、单一人群默认（亚裔），可切换但不强推

## 2. 技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 框架 | Svelte 5 + TypeScript | 编译时优化、bundle 小 |
| 构建 | Vite 5 | `npm run build` → `dist/` |
| 路由 | svelte-spa-router（hash 模式） | 3 个页面，hash 路由避开 GitHub Pages 路径配置 |
| 测试 | Vitest + @testing-library/svelte | 标准 Svelte 测试栈 |
| 部署 | GitHub Actions → `gh-pages` 分支 | push `main` 自动构建并部署，测试不通过则阻断 |

**不用 SvelteKit 的原因**：3 个页面 + hash 路由 ≈ 50 行路由代码，SvelteKit 的 SSR 收益为零，basePath / 404.html 配置反而是负担。

## 5. 数据模型

```ts
interface Measurements {
  height: number;  // cm, 100-250
  weight: number;  // kg, 20-300
  waist: number;   // cm, 40-200
}

type Ethnicity = 'asian' | 'non-asian';   // 默认 asian
type Diagnosis = 'underweight' | 'normal' | 'overweight' | 'obese';
```

## 6. 阈值

**亚裔（默认，2026 ADA 新标准）**

| 指标 | 正常 | 临界 | 异常 |
|---|---|---|---|
| BMI | 18.5 – 22.9 | 23.0 – 27.4 | ≥ 27.5 |
| 腰围（女 / 默认无性别） | < 80 | 80 – 89 | ≥ 80 |
| 腰围（男） | < 90 | 90 – 101 | ≥ 90 |
| WHtR | < 0.5 | — | ≥ 0.5 |

**非亚裔**

| 指标 | 正常 | 临界 | 异常 |
|---|---|---|---|
| BMI | 18.5 – 24.9 | 25.0 – 29.9 | ≥ 30 |
| 腰围（女 / 默认无性别） | < 88 | 88 – 101 | ≥ 88 |
| 腰围（男） | < 102 | 102 – 117 | ≥ 102 |
| WHtR | < 0.5 | — | ≥ 0.5 |

**腰围无性别时的默认策略**

- 用户不输入性别时，腰围阈值**采用较严的数值**（亚裔 80cm、非亚裔 88cm），对应女性标准
- 结果页加一行说明："腰围采用女性标准 (80cm)；男性实际阈值为 90cm"
- 理由：80–90cm 区间是男性最易漏诊的灰区，用严值更保守

## 7. 诊断逻辑

```ts
function diagnose(m: Measurements, e: Ethnicity = 'asian'): DiagnosisResult
```

**亚裔判定树**

```
BMI < 18.5                                  → underweight
18.5 ≤ BMI < 23                             → normal
23 ≤ BMI < 27.5
  └ WHtR < 0.5 AND 腰围 < 80                → overweight
  └ WHtR ≥ 0.5 OR 腰围 ≥ 80                 → obese
BMI ≥ 27.5                                  → obese
```

**非亚裔判定树**：BMI 切点改为 25 / 30，WHtR 仍是 0.5，腰围阈值改为上节表格中的对应值。

**结果对象**

```ts
interface DiagnosisResult {
  category: Diagnosis;
  metrics: {
    bmi: number;
    whtr: number;
    bmiLabel: '偏瘦' | '正常' | '超重' | '肥胖';
    whtrLabel: '正常' | '超标';
    waistLabel: '正常' | '超标';
  };
  triggers: string[];        // 例：['BMI 处于超重范围 (23–27.4)', '腰高比 0.51 ≥ 0.5']
  recommendation: string;    // 一句话行动建议
  disclaimer: string;        // 固定免责文案
}
```

**每个 category 的 recommendation 文案**：

| 分类 | 建议 |
|---|---|
| 偏瘦 | 关注营养摄入，必要时咨询医生评估是否存在吸收或代谢问题。 |
| 正常 | 维持健康饮食与规律活动即可。 |
| 超重 | 调整饮食结构（减少精制碳水、增加蛋白质与膳食纤维），增加日常活动量，并定期监测。 |
| 肥胖 | 建议前往医院体重门诊或内分泌科，结合血糖、血脂、肝功能等指标进行专业评估。 |

**输入框行为**

| 字段 | 类型 | 范围 | 步进 | 校验时机 |
|---|---|---|---|---|
| 身高 | number | 100–250 | 0.1 | blur + change |
| 体重 | number | 20–300 | 0.1 | blur + change |
| 腰围 | number | 40–200 | 0.1 | blur + change |

- 失焦时校验，不通过则在输入框下方红字提示
- 输入合法后实时清除错误
- 三个字段全部合法 → "开始测算"按钮启用
- 点击 → 调 `diagnose()` → 结果存入 store → 跳 `/#/result`

**亚裔/非亚裔切换**：不放在主表单，放在底部链接"非亚裔？切换人群"，点击展开一个 inline 单选。多数亚裔用户用不到。

**腰围测量提示**：折叠式，点击"展开"显示步骤说明（软尺 / 肚脐水平 / 呼气末 / 不要勒紧）。

## 11. 部署

**GitHub Actions 工作流**（`.github/workflows/deploy.yml`）

```yaml
on: push to main
jobs:
  build-and-deploy:
    - checkout
    - setup-node 20
    - npm ci
    - npm run build      # vite build → dist/
    - npm run test       # vitest run
    - upload dist/ to gh-pages branch
```

- 触发：push 到 `main` 即部署
- 测试不通过则阻断部署
- 首次需在 GitHub 仓库 Settings → Pages 选 `gh-pages` 分支

**构建配置**

- `vite.config.ts` 设 `base: './'`（相对路径，兼容 GitHub Pages 子路径与自定义域）
- 构建产物预期 < 50KB JS（gzip 后）

## 12. 测试

**必须覆盖**

| 范围 | 用例 | 数量级 |
|---|---|---|
| `diagnose.ts` 纯函数 | BMI 边界、WHtR 边界、组合触发、亚裔/非亚裔切换 | ~20 条 |

| InputField 组件 | 校验触发、错误展示 | ~3 条 |
| Result 组件（snapshot） | 各分类渲染快照 | 4 条 |

不写 E2E（Playwright 之类）—— 静态站点 + 纯函数，单元测试 + 手动视觉验证已足够。

**手动验收清单**

- [ ] Chrome / Safari / Firefox 桌面三浏览器

- [ ] 三个页面键盘导航可达
- [ ] 屏幕阅读器（VoiceOver / TalkBack）读出标签

## 13. 错误处理

| 场景 | 行为 |
|---|---|
| 输入非法 | 输入框红字提示，按钮禁用 |
| 直接访问 `/#/result` 但 store 为空 | 自动跳回 `/#/`，附 toast "请先完成测算" |
| 未知路由 | 跳 `/#/` |
| GitHub Pages 404（用户输入错误 URL） | 浏览器默认 404 页（不做自定义） |
| JS 加载失败 | `<noscript>` 提示 + body 兜底文案 |

## 14. 范围外（明确不做）

- 用户账号、历史记录、社交分享
- 国际化（仅中文）
- 体脂率 / 器官功能数据采集（需要医院设备）
- 国际化多人群切换之外的"中国旧标"独立模式（可在解读页内静态对比，不做交互式）
- PWA / 离线缓存
- 自定义域名（保持 `*.github.io`）
